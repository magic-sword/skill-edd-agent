# トリガー評価スキル

## 概要
このスキルは、指定されたスキルのトリガー定義の品質と、実際のトリガー精度を自動的に評価します。評価は以下の二段階ゲートで行われます。

1.  **第1ゲート (静的評価)**: スキルの `SKILL.md` を読み込み、トリガー条件の「具体性（Specificity）」と「明確性（Clarity）」を5点満点で評価します。両方とも4点以上でなければ不合格とし、評価を終了します。
2.  **第2ゲート (動的評価)**: 対象スキルがトリガーされるべき陽性プロンプト10件、およびトリガーされるべきではない陰性プロンプト10件の合計20件のテストケースを自動生成します。これらのテストケースを用いて、対象スキルが適切にトリガーされるか（陽性ケース）／されないか（陰性ケース）を検証し、そのパス率（精度）が90%以上（18/20件以上）であれば合格と判定します。

最終的な評価結果は中央レジストリに記録され、詳細レポートが生成されます。

## トリガー条件
以下のいずれかの表現でスキルがトリガーされます。

*   「[スキル名]のトリガーを評価して」
*   「[スキル名]のトリガー定義の品質を確認して」
*   「[スキル名]のトリガー精度をテストして」
*   「[スキル名]のトリガー評価を実行して」

## 使用手順
エージェントがこのスキルを実行するためのコマンドライン構文は以下の通りです。

```bash
python src/skills/trigger-evaluator/scripts/evaluate_trigger.py --skill_name [評価対象のスキル名（例: my-awesome-skill）]
```

## 権限レベル (Permission Level)
**3. Action-Allowed**

*   理由: このスキルは、ファイルシステムへの読み書き（`SKILL.md` の読み込み、`evalset.json` やレポートファイルの書き込み、レジストリファイルの更新）および外部API（Gemini API、AgentEvaluator）の呼び出しを伴うため、システムへの変更を許可する権限が必要です。

## 評価方法 (Evaluation Toolkit)

*   **Eval-as-Unit-Test**: この `trigger-evaluator` スキル自体の基本的な機能（引数解析、ファイル読み書き、ダミーAPI呼び出しのシミュレーション）が正しく動作するかを検証するために使用します。
*   **LLM-as-Judge**: 
    *   **静的評価**: 第1ゲートにおいて、Gemini APIを用いて対象スキルの `SKILL.md` のトリガー条件の「具体性」と「明確性」を評価します。LLMが評価基準に基づいてスコアリングを行います。
    *   **テストケース生成**: 第2ゲートにおいて、Gemini APIを用いて対象スキルのトリガー評価用の陽性・陰性プロンプトを自動生成します。これにより、多様なテストシナリオを網羅します。
*   **Golden Dataset**: 第2ゲートの動的評価において、自動生成されたテストケース（陽性・陰性プロンプト）がGolden Datasetとして機能し、対象スキルのトリガー動作の正否を判定します。

## 評価基準

*   **静的評価**: 具体性（Specificity）と明確性（Clarity）がそれぞれ5点満点中4点以上であること。
*   **動的評価**: 自動生成された20件のテストケースに対するトリガーパス率が90%以上であること。
